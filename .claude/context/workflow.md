# פורמט עבודה — Stocks Bank

> מדריך מעשי לעבודה יומיומית על הפרויקט. מיועד לצוות ללא רקע טכני עמוק.

---

## 🗺️ מפת המידע — מה נמצא איפה

| סוג מידע | מיקום | מי מעדכן |
|----------|-------|----------|
| **קוד האפליקציה** | Git (`src/`, `api/`) | מפתח (אייל/Claude) |
| **קבצי זיכרון משותפים** | Git (`.claude/context/`) | Claude (דרך `/update-memory`) |

**כלל ברזל:** נתונים דינמיים של מניות → Notion. כל השאר → Git.

## 📋 תזרים עבודה יומי

### 🌅 תחילת session — תמיד ככה מתחילים

```bash
# 1. ודא שאתה על main
git checkout main

# 2. משוך את הגרסה העדכנית ביותר מהצוות
git pull origin main

# 3. צור ברנץ' חדש למשימה
git checkout -b feature/שם-המשימה
```

**למה זה חשוב:** אם דילגת על `git pull`, אתה עובד על קוד ישן ותיצור קונפליקטים.

---

### 💻 במהלך העבודה — מחזור יומיומי

```bash
# הפעל שרת לוקלי
npm run dev
# → פתח http://localhost:5173 בדפדפן
```

**אתה עובד, מבצע שינויים, בודק שעובד בדפדפן...**

**כל כמה שעות או בסוף כל שלב לוגי:**
```bash
git add .
git commit -m "תיאור קצר של מה שינית"
```
זה **שומר נקודת שחזור מקומית** — לא מגיע לצוות עדיין.

---

### 🚀 סיום יום עבודה — push לענן

```bash
git push -u origin feature/שם-המשימה
```

**מה קורה אחרי push:**
1. הקוד עולה ל-GitHub
2. Vercel מזהה אוטומטית ובונה **Preview URL** (למשל: `stocks-bank-git-feature-xyz.vercel.app`)
3. גיא ורוני יכולים לראות ולהגיב

---

### ✅ סיום משימה — merge ל-main

**אופציה 1 — דרך GitHub (מומלץ):**
1. היכנס ל-https://github.com/uniquelyoursstore/stocks-bank
2. לחץ על "Pull Requests" → "New Pull Request"
3. בחר את הברנץ' שלך מול `main`
4. בקש אישור מגיא/רוני (אם צריך)
5. לחץ "Merge"
6. Vercel מפרסם אוטומטית לפרודקשן

**אופציה 2 — ישירות מהטרמינל (מהיר, למשימות קטנות):**
```bash
git checkout main
git pull origin main
git merge feature/שם-המשימה
git push origin main
```

## 🆘 מה עושים כשמשהו נשבר

### "עשיתי pull ויש קונפליקטים"
```bash
git status              # רואה אילו קבצים בקונפליקט
# תגיד ל-Claude: "יש לי קונפליקט בקובץ X, תעזור לי לפתור"
```

### "שיניתי משהו ושברתי הכל"
```bash
git checkout -- .       # מבטל שינויים שלא נשמרו (commit)
# או:
git reset --hard HEAD   # חוזר לcommit האחרון (זהירות! מוחק שינויים)
```

### "הברנץ' שלי מיושן מדי"
```bash
git checkout main
git pull origin main
git checkout feature/שלי
git merge main          # מביא שינויים מ-main לברנץ' שלך
```

### "עשיתי commit לברנץ' הלא נכון"
**תגיד ל-Claude** — זה מורכב, עדיף לא לנסות לבד.

## ⚠️ טעויות נפוצות שצריך להימנע מהן

1. ❌ **לעבוד ישירות על `main`** → תמיד ברנץ' נפרד
2. ❌ **לדלג על `git pull`** בתחילת היום → קונפליקטים
3. ❌ **לשכוח `/update-memory`** → אובדן קונטקסט
4. ❌ **שני אנשים על אותו קובץ** → merge conflicts
5. ❌ **לשכוח `git push`** לפני כיבוי המחשב → עבודה אבודה אם המחשב נדפק
6. ❌ **לערוך `public/alerts.json` ידנית** → השתמש ב-`/alert-add`
